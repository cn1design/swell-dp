# 新規実装：チュートリアル機能（ウォークスルー＋ヘルプボタン）

スプラッシュ画面の「さっそく使ってみる」を押した直後に開始される、画面ハイライト型のチュートリアル（ウォークスルー）機能と、再確認用のヘルプボタンを実装する。ライブラリ（Intro.js など）は使用せず、軽量な独自 JS/CSS で構築する。

---

## 0. 前提条件：スプラッシュ画面（Phase 1 で実装）

チュートリアルのエントリーポイントとなるスプラッシュ画面は**現在未実装**のため、本機能と同時に新規実装する。

### スプラッシュ画面の仕様

- **表示条件:** `localStorage.dp_tutorial_done` が存在しない初回訪問時のみ表示
- **構成:** ロゴ or サービス名 ＋ キャッチコピー ＋「さっそく使ってみる」ボタン
- **動作:**「さっそく使ってみる」クリック → スプラッシュをフェードアウト → チュートリアル Step 1 を起動
- **実装場所:** `archive-design_pattern.php` および `page-design_pattern_standard.php` の両ページに注入（PHP）
- **CSS:** `scss/_c-tutorial.scss` に記述
- **JS:** `javascript/dp-tutorial.js` に記述

## 1. チュートリアル（ウォークスルー）の仕様

特定の要素の `z-index` を上げてハイライトし、その付近に吹き出し（ツールチップ）を表示する。

### ステップ定義（全4ステップ）

| Step | ハイライト対象                                                         | ツールチップ位置 | テキスト                                                                          |
| ---- | ---------------------------------------------------------------------- | ---------------- | --------------------------------------------------------------------------------- |
| 1    | 共通 CSS コピーエリア（`#dp-base-css-area` 等）                        | 下               | 「まずはベースを整える『共通CSS』をコピーして、エディタの一番上に貼り付けます。」 |
| 2    | 先頭カードのチェックボックス（`.pl-card:first-child .pl-card-select`） | 右または下       | 「使いたいデザインが見つかったら、チェックを入れて追加します。」                  |
| 3    | LPビルダートリガーボタン（`#dp-cart-trigger`）                         | 上               | 「選んだパーツはここから確認できます。」                                          |
| 4    | 一括コピーボタン（`#dp-cart-copy-btn`）                                | 上               | 「順番を入れ替えて『一括コピー』を押し、エディタに貼り付ければ完成です！」        |

#### Step 4 の特別処理

1. JS がプログラムでカートドロワーを自動展開する（`openDrawer()` を呼ぶ）
2. カートが空の場合、チュートリアル専用のダミーアイテム（title: 「ヒーローセクション（例）」）を仮挿入し、コピーボタンの `disabled` を一時解除する
3. チュートリアル終了時（完了/スキップ）にダミーアイテムを除去し、ドロワーを閉じる
4. 吹き出しの上部に GIF 動画用プレースホルダー `div.tutorial-gif-placeholder` を設ける

### ツールチップ（吹き出し）の配置ロジック

- ターゲット要素の `getBoundingClientRect()` を取得し、**指定方向**（上記テーブルの「ツールチップ位置」列）に配置する
- 画面端にはみ出す場合は反対方向にフォールバックする
- SP（959px以下）では画面下部に固定表示（`position: fixed; bottom: 16px; left: 50%`）

### 吹き出し内の UI

- ステップ数表記（例：`1 / 4`）
- 「次へ」ボタン（最終ステップは「はじめる」）
- 「説明をスキップ」リンク（いつでもチュートリアルを終了）

### 動作の仕組み

- `body` にダークオーバーレイ（`z-index: 9000`）をかける
- ターゲット要素に `is-tutorial-highlight` クラスを付与し `z-index: 9001` でオーバーレイの上に浮き出させる
- ツールチップは `z-index: 9002`
- **Step 4 のみ**、カートドロワー全体（`#dp-cart-drawer`）に `is-tutorial-active` クラスを付与して `z-index: 9001` に昇格させる（ドロワー内の一括コピーボタンも同時に浮き出る）
- 「次へ」ボタンクリックのみで次ステップへ進む（オーバーレイクリックでの進行はしない）

## 2. ヘルプボタン（再確認用）の仕様

- 縦書き CTA ボタン（`.dp-vertical-right-ja`）の**上**に重ならない位置に固定配置する
  - PC: `position: fixed; right: 16px; bottom: 160px;`（既存 CTA ボタンより上）
  - SP: `position: fixed; right: 16px; bottom: 80px;`
- ラベル：「使い方」（アイコン ＋ テキスト）
- クリックするとウォークスルーが Step 1 から再起動する

## 5. 実装上の技術的注意事項（必読）

### 5-1. ターゲット要素への自動スクロール（Viewport 確保）

ユーザーの画面サイズによって Step 2（カード）や Step 3（トリガーボタン）がビューポート外にある場合がある。
**次のステップに進む際、ツールチップ位置計算の前に必ず以下を実行すること。**

```js
target.scrollIntoView({ behavior: "smooth", block: "center" });
// scrollIntoView 完了を待ってから getBoundingClientRect() を取得する
// → setTimeout または scrollend イベントで遅延を設ける
```

- `scrollIntoView` はアニメーションを伴うため、完了前に `getBoundingClientRect()` を取得するとズレる
- `setTimeout(fn, 400)` か `window.addEventListener('scrollend', fn, { once: true })` で待機してから位置計算を行う

---

### 5-2. スタッキングコンテキスト（z-index の罠）

ターゲット要素に `z-index: 9001` を付与しても、**親要素に `overflow: hidden` や独自の `z-index` が設定されている場合**、オーバーレイ（9000）の上に抜け出せず、ハイライトが切り取られる。

**対策（優先順）:**

1. ターゲット要素の親チェーンを遡り、スタッキングコンテキストを形成している要素を特定して `z-index` を一時昇格させる
2. 上記が困難な場合は **box-shadow ハック**を採用する:

```css
/* ターゲット要素に直接かけて疑似オーバーレイを作る */
.is-tutorial-highlight {
  position: relative;
  z-index: 9001;
  box-shadow: 0 0 0 9999px rgba(0, 0, 0, 0.6);
  border-radius: 4px; /* ターゲット形状に合わせて調整 */
}
```

- box-shadow ハックを使う場合、`body` 側のオーバーレイ要素は不要（または透明にする）
- Step 2（`.pl-card-select`）は `.pl-card-thumb`（`position: relative`）の内側にあるため、**`.pl-card-thumb` ごと昇格させる**必要がある

---

### 5-3. Step 4 ドロワーアニメーションの待機

`openDrawer()` を呼び出した直後にコピーボタンの座標を取得すると、スライドインアニメーション（350ms）の途中のズレた位置に吹き出しが表示される。

**必ず以下のいずれかで待機してからツールチップを描画すること:**

```js
// 方法A: transitionend イベント（推奨）
drawer.addEventListener(
  "transitionend",
  function handler() {
    drawer.removeEventListener("transitionend", handler);
    positionTooltip(copyBtn);
  },
  { once: true },
);
openDrawer();

// 方法B: setTimeout（フォールバック）
openDrawer();
setTimeout(function () {
  positionTooltip(copyBtn);
}, 400); // 350ms アニメーション + 50ms バッファ
```

- `transitionend` は複数プロパティ分発火するため `{ once: true }` または `e.propertyName === 'transform'` で絞り込む

## チュートリアルの動作確認結果

チュートリアルの動作確認を行いましたが、Step 3（LPビルダートリガーボタンのハイライト）で重大な表示不具合がありました。修正をお願いします。

## 【発生している問題】

1. LPビルダートリガーボタンは、スクロール連動で表示される仕様（初期状態は非表示/opacity:0 等）になっています。
2. チュートリアル実行時、ユーザーがスクロールしていない（初期状態の）場合、ターゲットであるボタンが非表示のままになっています。
3. 非表示のためJSでの座標計算（`getBoundingClientRect`など）が狂い、本来右下に出るべき吹き出しが、左下などの見当違いの場所に出現しています。
4. Step 3でボタンを認識できないままStep 4で突然ドロワーが開くため、ユーザーが困惑するUXになっています。

## 【修正要件（実装方針）】

**1. チュートリアル中のトリガーボタン強制表示**
チュートリアルが起動している間（またはStep 3のタイミング）は、スクロール位置に関わらずLPビルダートリガーボタン（`#dp-cart-trigger` 等）を強制的に画面に表示させる処理を追加してください。

- 解決案：チュートリアル実行中、`body` に `is-tutorial-running` などのクラスを付与し、CSSで `.is-tutorial-running #dp-cart-trigger { opacity: 1 !important; visibility: visible !important; transform: none !important; }` と強制上書きする。

**2. ハイライト時のボーダー（枠線）追加**
ハイライトされた要素がどこにあるか一目でわかるよう、`is-tutorial-highlight` クラスに対して、目立つ四角いボーダーを付与してください。

- レイアウト崩れを防ぐため、`border` ではなく `outline: 3px solid var(--color_main, #f00); outline-offset: 4px;` や、それに類する `box-shadow` を使用して、対象要素を四角く囲んで強調してください。

**3. 座標計算の再確認**
強制表示させた後であれば、要素の実態（幅・高さ・座標）が存在するため、Step 3のツールチップ（吹き出し）が正しくボタンの付近（上部など）に配置されるはずです。表示・計算のタイミングがズレないように留意してください。

上記3点を修正し、Step 3で「右下に強制表示されたボタンが四角くハイライトされ、正しい位置に吹き出しが出る」ことを確認してください。
