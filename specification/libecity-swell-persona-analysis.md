## 1. ユーザー属性の分類

投稿者を目的・スキル感で分類すると、大きく4タイプが確認できた。

| タイプ | 比率（推定） | 代表的な発言 |
|--------|------------|-------------|
| **A: ブロガー（収益化・アフィリエイト目的）** | 約40% | 「SWELLに乗り換えた」「記事リライト中」「テンプレが便利」 |
| **B: Web制作者（副業・フリーランス）** | 約30% | 「納品案件でSWELLを使い倒してる」「AI＋SWELLが最適解」 |
| **C: 学習中・初心者** | 約20% | 「操作が少しわかってきた」「夏にリニューアルしてみよう」 |
| **D: AIヘビーユーザー（実験・効率化志向）** | 約10% | 「CodexでSWELL実装を試した」「SWELL専用ブロックを使わせたい」 |

## 2. ペルソナ共通パーソナリティ

### 共通する行動・価値観

- **「やってみた報告」が多い** — 完成・完璧より「今日進んだ」を発信する文化。毎日の小さな進捗を積み上げている
- **SWELLを「自分で使いこなす」ことに誇りを感じている** — 「機能がわかってきた」「使い倒してる感覚がある」など達成感の表現が多い
- **AI活用に積極的・ただし"AIだけ"には距離感がある** — 「AIで作っただけのサイトじゃ出せない生っぽい感じを忘れずに」という発言が象徴的
- **情報共有・コミュニティ文化が強い** — CSSコードやTipsをタグ付き（#SWELLメモ）でシェアする習慣がある
- **時間・コストに敏感** — 「時短」「効率化」「コーディング時間短縮」が共通の関心事

### よく使われているキーワード
`時短` / `コピペ` / `使いやすい` / `慣れてきた` / `作り込める` / `AI＋SWELL` / `SWELL専用ブロック` / `テンプレ`

## 3. 発見したペインポイント（SWELL-DPが刺さるポイント）

### ペイン① 「AIはSWELL専用ブロックを使ってくれない」
> 「SWELL専用ブロックをできるだけ使うように指示したのに、標準のカラムとか使われるのが惜しい...」
> 「AIよ、あと、もう一歩！という感じです。」

→ **SWELL-DPはこの解決策そのもの。** AIが出力した結果がすでに「SWELL専用ブロック前提」になっている点が刺さる。

### ペイン② 「コーディング知識がないと細かい実装ができない」
> 「PHP触ってない」「CSS知識があれば作り込める」「追加CSS頼り」

→ ブロックコードをコピペするだけでデザインが完成する「ポータビリティ優先」の設計思想が直接響く。

### ペイン③ 「AIで雰囲気は作れるが、生感・クオリティが落ちる」
> 「AIで作っただけのサイトじゃ出せない生っぽい感じを忘れずに」

→ 人間が設計したパターンを使うことで「AIっぽさ」を回避できる点を訴求できる。

### ペイン④ 「実装の再現性が低い・毎回ゼロから考えている」
> 「細かい修正の指示とか大変だから理想の状態を一旦作ってもらって、それを自分でSWELLで再現が最適解」

→ 「パターンが決まっていれば迷わない」という訴求が響く。

## 4. SWELL-DP の訴求角度（優先順）

### 角度①【一番刺さる】「AIがSWELL専用ブロックを使えない問題」への直接回答
```
AIにSWELL専用ブロックを使わせたいですよね。
SWELL-DPは「最初からSWELL専用ブロックで書かれた」コードパターン集です。
AIではなく、人間が設計したブロックコードをそのままコピペするだけ。
```

**なぜ響くか：** リベシティのAI活用ユーザーが直接体験している不満への即答。「わかってる」感が信頼につながる。

---

### 角度②【広く刺さる】「SWELL、さらに使い込みたい人向け」
```
「SWELL使いやすいな」と感じてきた人へ。
次のステップは「デザインパターンを使い回すこと」。
LP・コーポレートサイトのセクションをコピペ1分で追加できます。
```

**なぜ響くか：** 「慣れてきた→もっと使いこなしたい」という成長欲求にフィットする。押しつけでなく「次のステップ」という提示が自然。

---

### 角度③【制作者向け】「納品スピードを上げたい」
```
SWELLで案件を受けるなら、パターン化しておくと強い。
ヒーロー、CTA、お客様の声……よく使うセクションをストックしておけば
毎回ゼロから組む必要がなくなります。
```

**なぜ響くか：** 「案件でSWELLを使い倒している」制作者への実務的な提案。ROIが見えやすい。

---

### 角度④【初心者向け】「まずはコピペから始めよう」
```
SWELLの操作まだ慣れてない…そんな時こそコピペが最速。
デザインされたブロックコードを貼るだけで、
プロっぽいセクションが完成します。
```

**なぜ響くか：** 「まだ慣れていない」ユーザーの心理的ハードルを下げる。「失敗しない」「難しくない」が刺さる。

## 5. NG訴求・注意点

| NG | 理由 |
|----|------|
| 「AIが自動生成」を前面に出す | AIへの不信感・「生っぽさが失われる」懸念と逆行 |
| 技術的すぎる説明（SCSS・PHPなど） | ブロガー層は技術用語にアレルギーがある |
| 「完璧なサイトが作れる」的な大げさな訴求 | リベシティ文化は「やってみた」の積み上げ。過剰な完成形より「始めやすさ」が響く |
| 一方的な宣伝・売り込み感 | コミュニティ内の投稿は「共有・シェア」文化。TIPS感・体験談形式が馴染む |

## 7. まとめ

リベシティのSWELLユーザーは「学びながら手を動かす、リアルな実践者」が中心。
宣伝より**体験談・TIPS・具体的な操作感**が刺さるコミュニティ文化を持つ。

SWELL-DPは「AIがSWELL専用ブロックを使わない問題」と「コピペで時短したい」という2つのペインに直接刺さるポジションにある。
訴求の軸は **「SWELL専用ブロック × コピペ完結 × 人間が設計」** の3点セットにするのが最も自然。
