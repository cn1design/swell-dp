# 以下は私の個人的な見解なので参考程度に見ておいてください。

■ UI/UX要件

コンポーネント配置: 共通の「カードコンポーネント」内の指定コンテナに対し、X（旧Twitter）ライクなハート型のトグルボタン（Likeボタン）をマウントすること。レイアウトの最適化も併せてよしなに調整をお願いします。

コールドスタート（初期0件）対策: リリース初期はLike数が0件となるため、社会的証明の欠如によるネガティブな印象（過疎感）を与えないフロントエンドの表示制御を組み込むこと。

要件例: count === 0 の場合は数値をDOMにレンダリングしない（非表示にする）、またはプレースホルダー的なアイコンのみの表示とするなど。

■ 機能・システム要件

非同期通信と楽観的UI（Optimistic Updates）: アクション（Click/Tapイベント）発火時、バックエンドへの非同期リクエスト（API Call）を行うと同時に、サーバーからのレスポンス完了を待たずにフロントエンドのカウントを即時インクリメント（+1）し、ユーザー体感を向上させること。

状態のリアクティブ同期: 該当のカードコンポーネントは、ルーティングやビューを跨いで最大3箇所にインスタンスがマウントされる仕様です。いずれかのコンポーネントでイベントが発火した際、グローバルステート（Redux, Context, Pinia等）のストアを更新、あるいはカスタムイベントを発火させるなどして、現在マウントされている全3箇所のコンポーネントのDOM（いいね数）をリアルタイムに同期・再レンダリングさせること。
