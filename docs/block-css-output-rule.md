あなたはWordPressテーマ「SWELL」の高度なコーディングアシスタントです。

現在、指定されたWebサイトからデザインパターンの生コード（HTML + SCSS/CSS）を取得し、長冗長な共通スタイルを削除した上で、ローカルのファイルを最適化されたコードで上書きする作業を行っています。

以下の【共通CSSルール】は、既に全ページ共通のベーススタイルとして読み込まれるため、各ブロックコードのインラインCSS（<style>タグ内）からは完全に削除する必要があります。

共通ルールを除外した上で、「そのパターン固有のレイアウトや装飾CSS」のみを残し、SCSSのネストを解いた標準のCSSとして出力先ファイルを上書きしてください。

【共通CSSルール（以下の記述は個別のHTMLからすべて削除すること）】

1. :root の変数定義

2. 見出し（h2, h3）の装飾リセット（border, background, padding, marginの初期化や、::before/::afterの content: none; position: unset; inset: unset; など）

3. 汎用テキストクラス（.c-lead, .c-copy, .c-summary, .c-pricing-number, .c-term, .c-annotation）

4. 汎用リストクラス（ul.-list-flex, .list-keyword, .list-hashtag とその子要素liのスタイル）

5. セクションタイトル（.dp-section-title, .dp-section-title--a, .dp-section-title--b とその内部のh2, p, span等のスタイル）

6. 表示切替ユーティリティ（.-sp, .-pc）

7. SWELL画像ブロックの pointer-events: none; などのリセット処理

【作業手順】

1. まず、`https://cn-design.biz/blog/design-patterns/` にアクセスし、ページソースを取得・解析してください。

2. 取得したソースから、対象のスラッグ（初回は `media-text-04`）を持つパターンアイテムを見つけてください。

3. そのアイテムの「コピーボタン」のデータ属性、または関連するスクリプト/隠し要素から「生のブロックコード（HTML + SCSS/CSS）」を抽出してください。

4. 抽出したコード内の <style> 〜 </style> を解析し、上記の【共通CSSルール】をすべて削除してください。

5. 残ったパターン特化のSCSSを、ネストのない標準的なCSSにコンパイルし、インデントをきれいに整形してください。

6. 現在のディレクトリにある出力先ファイル（初回は `media-text-04.html`）を、最適化されたHTML＋CSSのコードで【丸ごと上書き保存】してください。

7. 1件完了したら、変更内容の要約をターミナルに出力し、待機してください。私が「次へ」または「OK」と指示したら、続けて `media-text-05`, `mv-lp-01` の順番で対象スラッグを変更しながら、2〜6の手順を1件ずつ実行してください。

【注意事項】

- 共通ルールに該当するか迷った場合は、対象コンポーネントの「固有のクラス名」に紐づくスタイルは残してください。

- 出力は必ず標準のCSS（入れ子なし）とし、CSS変数はそのまま維持してください。

- <style>以外のHTMLブロック構造（DOM）は取得した生のコードのまま維持し、勝手に改変しないでください。

それでは、まず `media-text-04` の取得と `media-text-04.html` の上書き処理から開始してください。
